# Tag Vocabulary (v1)

> **Closed list, hierarchical.** Tags not present here are rejected by `lint_vault.py`. To extend: edit this file, bump the version below, push, and `git submodule update --remote` in every vault that uses it.

**Version:** 1.0.0
**Updated:** 2026-04-30

---

## Conventions

- Tags use forward slashes: `research/agents`, never `research-agents` or `research_agents`.
- Tags are lowercase, ASCII, hyphen-separated within a segment.
- A note can have multiple tags from different roots; e.g. `[research/agents, governance/eu-ai-act, access/public]`.
- Every note **MUST** carry exactly one `access/*` tag and exactly one `type/*` tag.
- Other roots are optional but encouraged.

## `workflow/*` — lifecycle state

Optional but recommended for entity / concept / wiki notes.

- `workflow/draft-llm` — generated by pipeline, not yet reviewed
- `workflow/candidate` — staged for review (concept candidates)
- `workflow/draft` — human draft, in progress
- `workflow/review` — ready for review
- `workflow/evergreen` — curated, blindado contra reescritura automática
- `workflow/archived` — superseded
- `workflow/deprecated` — explicitly retired, kept for history

## `access/*` — confidentiality (REQUIRED)

Exactly one per note. **Hard boundary.** A note tagged `access/confidential` MUST NOT live in any vault with a public or personal-cloud remote. Pre-commit hooks enforce this in `santander-kb`.

- `access/public` — fine to share, no restrictions
- `access/internal` — internal to a personal project, do not publish
- `access/confidential` — bank-internal or partner-confidential, regulated
- `access/restricted` — heightened restriction (legal hold, M&A, etc.)
